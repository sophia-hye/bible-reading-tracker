# Bible Reading Tracker 기획 문서

> **요약**: 성경 통독 트래킹을 엔진으로 삼는 **교회 커뮤니티 SNS 앱**(iOS/Android).
> **북극성은 "커뮤니티-first(SNS 중심)"** — 인스타그램처럼 **피드가 홈**이고, QT/묵상 나눔·통독 인증·기도제목·자유 게시글을
> 교회 그룹(닫힌 커뮤니티) 또는 공개로 나누며 서로 격려한다.
> 그 안에서 성경 통독 트래커(장별 체크, 비례 점 매트릭스 시각화)는 **매일 돌아오게 만드는 핵심 기능**으로,
> 1독으로 끝나지 않고 여러 회독(N독)을 누적하는 **"평생 통독" 여정**을 제공한다.
> **출시 전략**: 통독 트래커를 견고히 만든 뒤(MVP), 커뮤니티는 **소수 교회 비공개 베타**로 검증하고 공개로 확장한다.
> 여러 기기 간 로그인 동기화 지원.

## 1. 개요 (Overview)

### 1.1 목적 (Purpose)

**신앙 공동체가 함께 말씀을 읽고 나누는 디지털 공간**을 만든다. 핵심은 두 축이 맞물리는 선순환이다:
**① 커뮤니티(SNS)** — QT/묵상·통독 인증·기도제목·일상을 피드로 나누며(교회 그룹 또는 공개) 서로 격려·중보한다.
**② 통독 트래커(엔진)** — 매일 읽은 분량을 간편히 체크하고 진행률·streak·통계·성취·회독을 **시각적으로 채워지는 형태**로 확인,
이 통독 활동이 자연스럽게 피드의 인증 콘텐츠가 되어 커뮤니티를 살아 있게 한다.
혼자 하면 중도 포기율이 높은 통독을 **함께 읽는 공동체**로 끌어올려 완독률과 신앙 습관 지속을 돕는다.
여러 기기(아이폰/갤럭시/웹)에서 동일한 진행·피드를 동기화한다.

### 1.2 핵심 가치 제안 (Value Proposition)

| 키워드 | 설명 |
|--------|------|
| **완성감 (Sense of Completion)** | 1,189장이 하나씩 채워지는 시각화로 "내가 끝까지 가고 있다"는 성취감 제공 |
| **간편함 (Effortless)** | 탭 한 번으로 읽음 체크, 자동 진행률 계산, 오늘 읽을 분량 추천 |
| **동행 (Together)** | 친구·교회 그룹과 함께 읽고 격려, 리더보드로 선한 동기부여 |
| **꾸준함 (Consistency)** | streak, 알림, 챌린지, 배지로 매일 돌아오게 만드는 루프 |
| **확장되는 목표 (Expanding Goal)** | 1독 완료가 끝이 아니라 다음 회독 미션(3독·5독·10독…)으로 목표가 계속 확장 → "평생 통독" 여정으로 장기 리텐션 |
| **함께 나눔 (Community)** | QT/묵상·통독 인증·기도제목을 피드로 나누는 교회 SNS — 격려·중보로 신앙 공동체를 살아 있게 |
| **신뢰 공간 (Safe & Faithful)** | 교회 그룹 닫힌 커뮤니티 기본 + 공개 선택, 신고·차단·모더레이션으로 건강한 나눔 환경 보장 |

### 1.3 배경 (Background)

- 기존 성경 앱은 읽기(본문) 기능 중심이고 "통독 진행 관리"는 부가 기능에 그치는 경우가 많음
- 종이 통독표는 기기 분실/중복 체크 어려움, 통계·동기부여 부재
- 통독은 혼자 하면 중도 포기율이 높음 → 교회/소그룹의 **함께 읽기**가 완독률을 크게 높임
- 가족/소그룹이 여러 기기를 쓰는 환경에서 진행률 공유 필요 → 로그인 기반 클라우드 동기화 요구

### 1.4 페르소나 (Personas)

| 페르소나 | 상황 | 핵심 니즈 |
|----------|------|----------|
| 개인 통독러 (P1) | 올해 처음 1독에 도전 | 매일 분량 추천 + 진행률 시각화 + 알림 |
| 교회 소그룹 리더 (P2) | 청년부 20명과 90일 통독 + 나눔 운영 | 그룹 생성, 진행 현황 한눈에, 그룹 피드로 독려·모더레이션 |
| 함께읽는 친구 (P3) | 친구와 서로 자극받으며 읽음 | 팔로우, 통독 인증·QT 나눔, 리액션 |
| 복귀 독자 (P4) | 중간에 멈췄다가 다시 시작 | 어디까지 읽었는지 즉시 파악, 재시작 추천 |
| QT 나눔러 (P5) | 매일 묵상을 글/사진으로 기록·공유 | QT 작성, 그룹/공개 선택 공유, 댓글·중보 |
| 중보 기도자 (P6) | 공동체 기도제목을 챙김 | 기도제목 피드, 중보(기도했어요) 리액션, 알림 |
| 교회 관리자 (P7) | 교회 공식 계정·다중 그룹 운영 | 멤버 초대, 콘텐츠 모더레이션·신고 처리, 공지 |

## 2. 범위 (Scope)

> **제품 북극성(커뮤니티-first)**: 최종 형태는 **피드가 홈인 교회 커뮤니티 SNS**이고, 통독 트래커는 그 안의 핵심 엔진이다.
> 다만 **실행 순서는 리스크를 고려해 단계화**한다 — 먼저 통독 트래커를 견고히 출시(MVP)해 매일 쓰는 습관 루프를 검증하고,
> 커뮤니티(SNS)는 **소수 교회 비공개 베타(Phase 2)**로 안전성·모더레이션을 다진 뒤 **공개로 확장(Phase 3)**한다.
> 즉 비전은 커뮤니티 중심, 출시 경로는 트래커 → 비공개 베타 커뮤니티 → 공개.

### 2.1 MVP (1차 출시 — 통독 트래커 엔진)

- [ ] 성경 66권 / 1,189장 기준 읽기 진행률 추적 (장별 체크/해제)
- [ ] **회독(N독) 누적 추적** — 장별 "읽은 회차" 기록, 현재 회차 1,189장 완성 시 1독 확정 후 다음 회차 자동 시작 (회차별 기록 보존)
- [ ] **회독 미션 사다리** — 1독 우선, 완료 시 다음 목표(3독·5독·7독·10독…) 자동 제시 + 사용자 커스텀 목표 설정
- [ ] **누적 회독 시각화** — 점 매트릭스를 회독 깊이(읽은 회차 수)에 따라 농도/링으로 표현, 현재 회차 / 전체 누적 보기
- [ ] **앱 내 성경 본문 읽기** — 개역한글(KRV) 우선 제공, NIV는 라이선스 확보 후 추가 (번역본 전환)
- [ ] **시각적 진행 트래킹**: 책별 진행 바, 전체 그리드(1,189칸) 히트맵, 원형 진행 게이지
- [ ] 자동 진행률 계산 (전체 %, 책별 %, 신약/구약 구분)
- [ ] 성경 권별 통계 (완독 책 수, 책별 읽은 장/남은 장)
- [ ] 통독 계획 템플릿 선택 (1년 통독 / 90일 통독 / 신약 90일 등) 및 일자별 분량 자동 배정
- [ ] **통독 순서 추천** — 카테고리(성경 순서 / 주제별 / 신약 먼저 등) 선택 후 사용자 지정
- [ ] **오늘 읽을 분량 추천** (선택한 계획 + 현재 진행 기반)
- [ ] 일일 읽기 체크 + 연속 읽기 기록(streak) + 전체 진행률(%)
- [ ] 성취 데이터 / 배지 (첫 1장, 7일 연속, 첫 책 완독, 신약 완독, 1독 완료 등)
- [ ] 매일 읽기 리마인더 푸시 알림 (시간 설정 가능)
- [ ] 회원가입 / 로그인 (이메일, Apple, Google)
- [ ] 여러 기기 간 자동 동기화 (오프라인 우선 + 온라인 동기화)
- [ ] 오프라인 사용 (네트워크 없이 체크 → 재연결 시 동기화)
- [ ] 다크 모드 / 한국어·영어 UI

### 2.2 Phase 2 (교회 커뮤니티 SNS — 비공개 베타)

> 소수 교회를 초대해 **닫힌 베타**로 운영. 안전성(모더레이션·신고)·프라이버시·콘텐츠 품질을 검증한 뒤 Phase 3 공개.
> 이 단계에서 앱의 **홈이 피드로 전환**(커뮤니티-first)되고, 트래커는 탭/기능으로 자리한다.
> **전환 UX(FR-45)**: 기존 MVP 사용자에게는 홈 변경을 1회 안내 시트로 알리고, 커뮤니티 기능은 **옵트인**(원치 않으면 트래커 중심 사용 유지) 정책으로 이탈을 방지한다. 그룹 미가입 신규 사용자는 빈 피드 대신 공개 샘플/추천 게시물을 노출한다.

**그룹/소셜 기반**
- [ ] 교회/소그룹 생성·참여 (초대 코드/링크), 멤버 역할(관리자/멤버)
- [ ] 팔로우/팔로워, 사용자 프로필(통독 현황·게시물·배지)
- [ ] 그룹 진행 현황 대시보드 (멤버별 진행률, 그룹 누적 달성률)

**피드 & 게시물 (인스타그램형)**
- [ ] 홈 피드 (그룹 피드 + 팔로잉/공개 피드), 무한 스크롤
- [ ] 게시물 작성 4종: **① QT/묵상 나눔 ② 통독 진도/완독 인증 카드 ③ 자유 게시글+사진 ④ 기도제목/중보기도**
- [ ] **게시물별 공개 범위 선택**: 특정 그룹 / 공개(public) / 비공개(나만)
- [ ] 이미지 업로드(다중), 말씀 구절 인용, 통독 인증 카드 자동 생성(진행/회독/streak)
- [ ] 좋아요/리액션(아멘·기도했어요 등), 댓글, 멘션
- [ ] 알림 (댓글·리액션·팔로우·중보·그룹 활동)

**안전/운영 (베타 필수)**
- [ ] 신고·차단, 콘텐츠 모더레이션(관리자 큐), 욕설/유해 콘텐츠 1차 필터
- [ ] 공동체 가이드라인·신앙 콘텐츠 정책, 미성년 보호 정책

### 2.3 Phase 3 (공개 확장 & 심화)

- [ ] 공개 커뮤니티 전면 오픈(검색·탐색·추천 피드, 해시태그/주제)
- [ ] 기간형 챌린지 (90일/365일 통독) + 리더보드 (개인/그룹/public)
- [ ] 그룹 채팅, 공지, 이벤트
- [ ] 맞춤형 통독 계획 직접 생성, 통계 심화(연도별 회고 리포트)
- [ ] 위젯/잠금화면 진행률, Apple Watch·Wear OS 연동
- [ ] 결제 / 프리미엄 (광고 제거, 고급 통계, 교회 운영 도구 등)

### 2.4 Out of Scope (제외)

- NIV 본문 제공 — **라이선스(API.Bible/Biblica) 확보 전까지 제외**, 확보 후 추가 (개역한글은 MVP 포함)
- 오디오 성경 재생
- 실시간 영상/라이브 방송, DM(1:1 다이렉트 메시지) — Phase 3 이후 검토
- 결제 / 유료 구독 (Phase 3 이후 검토)
> 주의: 기존 "공개 SNS 피드 제외" 조항은 **본 개정으로 철회** — 공개 피드는 Phase 3 정식 범위(단, 모더레이션·신고 체계 확립 전제).

### 4.2 피드백·연출

- 장 체크 시 마이크로 인터랙션(채워짐 애니메이션 + 햅틱)
- 책 완독/배지 획득 시 축하 모달(컨페티, 공유 버튼)
- 진행 농담(명도)은 누적에 따라 단계적 변화로 "성장" 체감 (무채색 톤 유지, 4.3 참조)

### 4.3 비례 점 매트릭스 통독표 (Proportional Dot Matrix) — 핵심 아트 컨셉

> 스위스 스타일 **비례 점 매트릭스(proportional dot matrix)** 인포그래픽을 통독표에 적용.
> **행 = 성경 책, 열 = 장 번호**, **점 크기 = 장 길이(절 수)**, 읽은 장은 잉크 검정으로 채운다.
> 무채색·여백 중심의 미니멀 에디토리얼 미학으로 브랜드(무채색·미니멀)와 일치.
>
> **레퍼런스 미학**: ① 행=카테고리 / 열=인덱스 매트릭스, ② 점 크기로 양적 정보 인코딩,
> ③ 극도의 흑백 미니멀 + 충분한 여백, ④ 작은 산세리프 라벨(좌측 행명·하단 열번호).

**적용 매핑**

| 레퍼런스 | 통독표 적용 |
|----------|------------|
| 행(카테고리) | 성경 책 (책마다 장 수 달라 행 길이가 들쭉날쭉 → 책 길이가 한눈에) |
| 열(인덱스) | 장 번호 (하단 5·10단위 라벨) |
| 점 크기 = 데이터값 | **점 크기 = 장 길이(절 수)** — 큰 점=긴 장, 오해가 아니라 정보 |
| 점 채움 | 읽음 = 잉크 검정 / 안 읽음 = 연회색 |

**색/톤 전략 (확정): 무채색 모노톤 잉크**

- 통독표 배경은 **종이색(off-white)** 고정 → 잉크 점 대비 확보(라이트·다크 모드 무관)
- 읽은 장 = **잉크 검정**, 안 읽은 장 = **연회색** (이진). 다색은 쓰지 않음
- 진행은 **검정 채움이 행 단위로 번지며** 표현 → 한 책을 다 읽으면 그 행이 검게 완성
- (선택) 완성 임박 시 농담에 방향성을 약하게 더해 "그라데이션 흐름"을 줄 수 있음 — 구현 단계 미세조정

**화면 단위: 장르별**

오경/역사서/시가서/대선지서/소선지서(구약) + 복음서/역사서(행)/바울서신/일반서신/예언서(신약) 단위로 한 매트릭스.
(현 Figma 'Tracker' 디자인의 장르별 분할과 동일 — 시편 150장처럼 긴 책도 폰 화면에 수용)

**설계 원칙**

| # | 원칙 | 이유 |
|---|------|------|
| 1 | **행=책 / 열=장 매트릭스** (장르별 화면) | 성경 구조 그대로 매핑, 진행·위치 파악 즉시 |
| 1b | **모바일은 책별 장을 N열(기본 10)로 줄바꿈(wrap)** | 시편(150장) 등 긴 책의 **가로 스크롤 방지** → 세로 스크롤. 모바일 10열 / 태블릿·가로 15~20열 (반응형) |
| 2 | **점 크기 = 장 길이(절 수) 인코딩** | 레퍼런스 정신, 크기가 정보가 됨. 데이터 기반이라 항상 동일(결정론적) |
| 3 | **5·10장 단위 열 번호 라벨** (또는 wrap 줄 자체가 10장 눈금) | 장 번호를 빠르게 세고 위치 찾기 |
| 4 | **읽음 = 잉크 검정 / 안 읽음 = 연회색**, 종이색 배경 | 무채색 미니멀, 채움이 행 단위로 번짐 |
| 5 | **면적 착시 보완** | 점 크기는 장 길이일 뿐 진행량 아님 → 진행률 %·읽은 장 수를 숫자로 병기 |
| 6 | **터치는 시각과 분리 (히트박스 확대)** | 짧은 장(작은 점)도 투명 탭 영역을 크게 → 접근성·사용성 |
| 7 | (선택) **완독 포스터/공유 카드** | 다 채운 검은 매트릭스에 여백·타이포 얹어 소장·공유용 작품화 |

**모드 분리(선택지)**: 트래킹 가독성이 더 필요하면 "균일 그리드(1,189칸 히트맵)"와 "매트릭스 아트 모드"를 토글로 제공.

**절(verse) 단위 정밀 트래킹 (Phase 2 목표 — MVP는 장 단위로 완성)**: 장 단위가 거칠어 **절까지** 체크(FR-42).
- **앱 = 드릴다운**: 장 보기 점 → 한 장 탭 시 그 장의 **절 그리드**(1 dot = 1 verse)로 진입해 절 단위 체크.
- **인쇄 = 절 점 + 장 구분**: 1 dot = 1 verse, 장별로 묶어 표시(무채색, 1도 인쇄).
- **전제 데이터**: 장별 절 수 데이터셋(성경 총 약 31,102절) 필요 → 현 시안은 근사값, 실제 적용 시 정확 데이터로 교체.
- **번역본별 절 수 차이 + 온보딩 선택**: 개역한글과 NIV는 절 구분이 달라 절 수가 다름(예: 개역한글 ~31,102 / NIV ~31,086). **첫 설정(온보딩 Step 1)에서 번역본을 선택**하고, 그 번역본의 절 수 데이터(`verse.versionId`별)로 절 동그라미 개수를 생성. 설정에서 변경 가능. (MVP=개역한글, NIV=라이선스 후 "곧 제공")
- (대안) 장 점을 절 비율로 부분 채움하는 방식(A안)도 가능 — 점 1,189개 유지하면서 절 정밀도.

> 디자인 참조: 사용자 제시 레퍼런스(스위스 비례 점 매트릭스 인포그래픽) + Figma 'Tracker' 페이지 — [[figma-tracker-design]] (장르별 장 그리드 + 절 단위 드릴다운/인쇄 시안, 흑백 미니멀).

### 4.4 회독(Multiple Read-Throughs) — 평생 통독 여정 (핵심 컨셉)

> 본 앱의 궁극 목표는 **1회 통독이 아니라 여러 회독을 누적**하는 것. 1독 완료가 "끝"이 아니라
> 다음 미션의 "시작"이 되어, 사용자가 평생에 걸쳐 목표를 확장해 나가도록 설계한다.

**핵심 모델 (확정)**

| 항목 | 결정 |
|------|------|
| **카운팅 방식** | **회차별 순차 트래킹** — 현재 회차(cycle)의 1,189장을 모두 읽으면 **1독 확정** 후 다음 회차 자동 시작. 회차별 읽기 기록을 **보존**(과거 회독 회고 가능) |
| **미션 사다리** | **고정 사다리(1→3→5→7→10독…) + 커스텀 허용** — 1독을 최우선 목표로 두고, 완료 시 다음 단계를 자동 제시하되 사용자가 직접 목표 회차(예: "올해 2독")를 설정 가능 |
| **시각화** | **누적 — 농도/링으로 깊이 표현** — 점 매트릭스에서 한 장의 "읽은 회차 수"가 깊을수록 점이 진해지거나 링이 쌓여, 한 화면에서 "몇 회독째 깊이"를 한눈에. 무채색 미니멀 톤 유지(농담 단계로 표현). 현재 회차만 보기 / 전체 누적 보기는 추후 토글 검토 |

**동작 흐름**

```
1독 진행(현재 회차) → 1,189장 완성 → "1독 완료!" 축하 연출 + 배지
   → 다음 미션 제시(예: 3독 도전) → 회차 2 시작(매트릭스 농도 1단계 누적 시작)
   → … 목표 회차 도달 시 상위 미션으로 계속 확장
```

**설계 시사점 (설계 문서에 반영)**

- `reading_log`에 **회차(cycle) 차원 추가** → 키 = (user, cycle, book, chapter). 장별 "읽은 회차 수" = 해당 장이 read된 회차 개수.
- **완독 회차 수** = 모든 장이 채워진 회차의 개수. **현재 회차** = 완독 회차 + 1.
- 시각화 농도 = 장별 누적 읽은 회차 수 → 단계 매핑(무채색 농담).
- 미션/목표는 별도 엔티티(`reading_mission`)로 관리, 배지(5.1)와 연동.

### 4.5 커뮤니티 & 피드 (Community & Feed) — 인스타그램형 교회 SNS

> **북극성 기능**. 최종 형태에서 **피드가 홈**이며, 통독 트래커 활동이 인증 콘텐츠로 흘러들어 커뮤니티를 살아 있게 한다.
> Phase 2 비공개 베타로 시작 → Phase 3 공개 확장.

**게시물 타입 (4종)**

| 타입 | 설명 | 특수 요소 |
|------|------|----------|
| QT/묵상 나눔 | 오늘의 말씀 묵상·적용 글 | 말씀 구절 인용(책·장·절 메타), 본문 카드 |
| 통독 인증 카드 | 진도·회독·streak 달성 인증 | 트래커 데이터로 **카드 자동 생성**(점 매트릭스/링) |
| 자유 게시글 | 일상·교제·사진 | 다중 이미지 업로드 |
| 기도제목/중보 | 기도 요청·중보 | "기도했어요" 중보 리액션·중보자 수 |

**공개 범위 모델 (게시물별 선택)**

| 범위 | 노출 | 비고 |
|------|------|------|
| 그룹(group) | 선택한 교회/소그룹 멤버만 | 기본값 권장(닫힌 커뮤니티) |

| 비공개(private) | 본인만 | 개인 기록용 |

**피드 구성**: ① 그룹 피드(내 그룹들) ② 팔로잉 피드 ③ (Phase 3) 공개/탐색 피드. 무한 스크롤, 최신순 기본(추후 추천 랭킹).

**상호작용**: 좋아요/리액션(아멘·은혜·기도했어요), 댓글, 멘션, 팔로우, 알림.

**프로필**: 사용자별 통독 현황(진행률·회독·배지) + 게시물 그리드 → "신앙 여정 프로필".

**안전/운영 (베타 게이트)**

| 항목 | 내용 |
|------|------|

| 복음서 통과 | 4복음서 완독 | 진행 |
| 신약 완독 | 신약 27권 완독 | 진행 |
| 통독 완성 | 성경 1독 완료 | 정상 |

| 함께의 힘 | 그룹 통독 완주 | 소셜 |

### 5.2 챌린지 (Challenges)

| 챌린지 | 기간 | 일 분량(개략) | 비고 |
|--------|------|--------------|------|
| 90일 통독 | 90일 | 약 13~14장/일 | 집중 통독 |
| 365일 성경 읽기 | 1년 | 약 3~4장/일 | 가장 보편적 |
| 신약 90일 | 90일 | 약 3장/일 | 입문자용 |
| 시편/잠언 한 달 | 30일 | 약 5장/일 | 단기 도전 |

- 챌린지는 개인 단독 또는 그룹 단위 참여 가능
- 진행 상황은 챌린지 전용 진행 바 + 리더보드로 표시

### 5.3 리더보드

- **공개 범위(사용자 선택)**:
  - **개인용(Private)** — 나만 보는 개인 기록/순위 (기본값)
  - **그룹 공개(Group)** — 내가 선택한 특정 그룹에만 공개
  - **전체 공개(Public)** — 앱 전체 공개 리더보드에 노출
- 기준(선택): 누적 읽은 장, 현재 streak, 챌린지 진행률
- 정책: **경쟁보다 격려** — 기본값은 비공개(Private), 공개는 명시적 옵트인, 하위권 압박 최소화(예: 본인 ±N명만 표시 옵션)

### 5.4 일일 복귀 루프 (Retention Loop)

알림 → 오늘 분량 추천 → 탭 체크(완성감 연출) → streak·배지 업데이트 → 친구/그룹 활동 확인

## 6. 추천 로직 — 통독 순서 & 오늘 읽을 분량 (Recommendation)

### 6.0 통독 순서 카테고리 (Reading Order)

사용자가 **카테고리를 선택**하면 그에 맞는 읽기 순서/계획을 추천하고, 이후 **사용자가 직접 조정(사용자 지정)** 가능.

| 카테고리 | 설명 | 예시 순서 |
|----------|------|----------|
| 성경 순서 (정경순) | 창세기→요한계시록 그대로 | 전통적 통독 |
| 신약 먼저 | 신약 27권 후 구약 | 입문자/복귀자 추천 |
| 주제별 | 주제(은혜/지혜/예언/복음 등) 묶음으로 | 시편·잠언, 4복음서 등 테마 |
| 역사 흐름순 | 사건 연대기 순서 | 역사 이해 중심 |
| 장르별 | 율법/역사서/시가서/예언서/복음서/서신서 | 문학 유형별 |

- 카테고리 선택 → 추천 계획 미리보기 → 시작일·기간 조정 → 확정
- 확정된 순서는 `user_plan.schedule`로 저장, 오늘 분량 추천(6.1~6.3)의 입력이 됨

### 6.1 입력

- 선택한 통독 계획(총 일수, 일자별 분량표) 또는 챌린지
- 현재까지 읽은 장 / 마지막 읽은 위치
- 계획 시작일·오늘 날짜 (밀린 분량 계산)

### 6.2 규칙(초안)

1. **계획 기반**: 계획표의 "오늘 해당 일자" 분량을 기본 추천
2. **따라잡기(catch-up)**: 밀렸다면 남은 일수에 남은 분량을 재분배해 "오늘 권장량" 산출(과도하면 상한 적용)
3. **앞서감(ahead)**: 앞서 있으면 "오늘은 가볍게 / 다음 분량 미리읽기" 옵션 제시
4. **계획 미선택 시**: 균등 분배(예: 1년=하루 약 3.3장) 기반 다음 미읽은 구간 추천

### 6.3 출력

- "오늘 읽을 분량" 카드 (예: 마태복음 5~7장) + 예상 소요 시간 + 바로 체크 버튼

## 7. 데이터 모델 개요 (Data Model — 초안)

> 상세 스키마는 설계 단계에서 확정. 오프라인(SQLite) ↔ 서버(BaaS) 동기화 대상.

| 엔티티 | 핵심 필드 | 설명 |
|--------|----------|------|
| `user` | id, email, displayName, photo, locale, preferredVersion, leaderboardVisibility(private/group/public), createdAt | 계정 + 기본 번역본·리더보드 공개 설정 |
| `book` (정적) | id, name, testament, order, chapterCount, genre | 성경 66권 메타 (장르=율법/역사/시가/예언/복음/서신) |
| `bible_version` (정적) | id, code(KRV/NIV), name, source(local/api), licenseNote | 번역본 메타 (개역한글=로컬 번들, NIV=API.Bible) |
| `verse` (정적/캐시) | versionId, bookId, chapter, verse, text | 본문 (개역한글 번들; NIV는 라이선스 후 API 캐시) |
| `reading_log` | id, userId, **cycle**, bookId, chapter, readAt, source, updatedAt | 장별 읽음 기록(체크). **회차(cycle) 차원 포함** — 키=(user, cycle, book, chapter) |
| `reading_cycle` | userId, currentCycle, completedCycles, startedAt, updatedAt | 사용자별 현재 회차·완독 회차 수 (회독 진행 상태) |
| `reading_mission` | id, userId, targetCycle, type(ladder/custom), status, achievedAt | 회독 미션(목표 회차) — 사다리 단계 또는 커스텀 목표 |
| `reading_plan` (정적/템플릿) | id, name, totalDays, schedule(JSON) | 통독 계획 템플릿 |
| `user_plan` | id, userId, planId, startDate, status | 사용자가 선택한 계획 |
| `streak` | userId, current, longest, lastReadDate | 연속 기록 |
| `achievement` | id, userId, badgeKey, achievedAt | 획득 배지 |
| `challenge` | id, type, title, startDate, endDate, rule | 챌린지 정의 |
| `challenge_member` | challengeId, userId, progress, joinedAt | 챌린지 참여 |
| `group` | id, name, ownerId, inviteCode, planId | 교회/소그룹 |
| `group_member` | groupId, userId, role(admin/member), joinedAt | 그룹 멤버 |
| `follow` | followerId, followeeId, createdAt | 팔로우 관계 |
| `post` | id, userId, type(qt/cert/free/prayer), visibility(group/public/private), body, verseRef(JSON), cardMeta(JSON), prayerStatus(open/answered, prayer만), expiresAt(nullable), createdAt, updatedAt | 피드 게시물(4종). **groupId 제거** → 다중 그룹 공유는 post_group으로(DM-1) |
| `post_group` | postId, groupId | 게시물↔그룹 N:N (다중 그룹 동시 공유, DM-1) |
| `post_media` | id, postId, url, thumbUrl, order, width, height | 게시물 이미지(Supabase Storage) |
| `comment` | id, postId, userId, body, parentId(nullable), createdAt | 댓글. **대댓글 깊이 1단계 제한**(parentId의 parentId는 null, DM-2) |
| `reaction` | id, targetType(post/comment), targetId, userId, kind(like/amen/pray), createdAt | 좋아요/리액션·중보. kind='pray'=중보(DM-4). targetId 존재검증은 앱 레이어 강제 |
| `notification` | id, userId, type, actorId, targetType, targetId, read, createdAt | 인앱 알림 |
| `user_push_token` | id, userId, token, platform(ios/android), createdAt, updatedAt | 기기별 Expo 푸시 토큰(DM-3) |
| `user_notification_pref` | userId, type, pushEnabled, inAppEnabled | 카테고리별 알림 설정(DM-3, FR-43) |
| `user_device` | id, userId, deviceId, platform, label, lastSeenAt | 기기 관리·원격 로그아웃(DM-7, FR-46) |
| `report` | id, reporterId, targetType, targetId, reason, status, createdAt | 신고 |
| `block` | userId, blockedUserId, createdAt | 차단. 피드 쿼리 차단 필터 성능은 설계서에서 전략 확정(DM-5) |

## 10. 아키텍처 고려사항 (Architecture Considerations)

### 10.1 프로젝트 레벨 선택

| Level | 특징 | 적합 대상 | 선택 |
|-------|------|----------|:----:|
| **Starter** | 단순 구조, 백엔드 없음 | 정적 사이트, 포트폴리오 | ☐ |
| **Dynamic** | 기능 모듈 구조 + BaaS 통합 | 백엔드 있는 앱, SaaS MVP | ☑ |
| **Enterprise** | 엄격한 레이어 분리, 마이크로서비스 | 대규모 트래픽 | ☐ |

> **선택: Dynamic** — 로그인 + 멀티 기기 동기화 + 커뮤니티 SNS(피드·이미지·모더레이션)가 필요하므로 BaaS 기반 풀스택 구조가 적합.
> 단, 커뮤니티-first로 확장되며 **UGC·피드 스케일·모더레이션** 요구가 커지므로, Phase 3 공개 시점에 Enterprise 수준(집계 캐시·CDN·콘텐츠 안전 파이프라인) 재평가가 필요하다.

### 10.2 핵심 아키텍처 결정

| 결정 항목 | 옵션 | 선택 | 근거 |
|----------|------|------|------|
| 앱 프레임워크 | React Native(Expo) / Flutter / PWA | **React Native + Expo** | 코드 하나로 iOS/Android, 푸시·오프라인 강점, TS 생태계 |
| 언어 | TypeScript / JavaScript | **TypeScript** | 타입 안정성 |
| 상태관리 | Context / Zustand / Redux | **Zustand** | 경량, 보일러플레이트 적음 |
| 로컬 저장(오프라인) | SQLite / AsyncStorage / WatermelonDB | **SQLite (expo-sqlite)** | 구조적 데이터, 오프라인 우선 |
| 백엔드(BaaS) | Supabase / Firebase / bkend.ai | **확정: Supabase** | Postgres + Auth + 실시간(그룹/리더보드) + RLS, 무료 티어 |
| 인증 | BaaS Auth (이메일+소셜) | **BaaS 내장 Auth** | 구현 비용 절감 |
| 푸시 알림 | Expo Notifications | **Expo Notifications** | 크로스플랫폼 통합 |
| 차트/시각화 | victory-native / react-native-svg / skia | **react-native-svg (+ Skia 검토)** | 원형 게이지·그리드 커스텀 렌더 |
| 네비게이션 | Expo Router / React Navigation | **Expo Router** | 파일 기반 라우팅 |
| 이미지 저장 | Supabase Storage / S3 / Cloudinary | **Supabase Storage (+ CDN)** | BaaS 일원화, 서명 URL·RLS 연동. 업로드 크기/형식 제한 |

| 미디어 처리 | 클라 리사이즈 / 서버 | **클라 리사이즈·압축 후 업로드** | 비용·성능, 썸네일 생성 |
| 테스트 | Jest / Vitest / Playwright | **Jest + RN Testing Library** | RN 표준 |

> 소셜·피드 데이터 접근 제어는 BaaS의 Row Level Security(RLS) + 그룹 멤버십/팔로우/차단 조인으로 가시성(그룹/공개/비공개)을 강제한다.

### A.1 개역한글 (KRV) — 대한성서공회

- **저작권료**: 무료. 저작재산권 보호기간(50년) 경과로 저작권료 지급 없이 사용 가능.
- **준수 조건 (인격저작권)**:
  - **동일성유지권**: 본문 내용·형식·제호 무단 변경/삭제/개변 불가 → 앱에서 본문을 **수정 없이 원문 그대로** 제공해야 함.
  - **성명표시권**: 저작자/출처 표기 준수.

### A.2 NIV — Biblica / API.Bible

- **저작권**: 보호 대상. 무단 사용 불가.
- **라이선스 경로**: **API.Bible**(scripture.api.bible)를 통해 번역본 단위 라이선스.
  - 비상업 Starter: 무료(월 5,000 콜, 일부 번역본).
  - 상업적 사용: 번역본당 **월 $10~** + Pro 플랜(월 150,000 콜 수준). "상업적 사용"에는 광고·인앱·구독·유료 접근 등 수익화 일체 포함.
  - 비인쇄/디지털 상업 사용은 **Biblica 별도 허가 신청** 필요(일부 royalty-free 라이선스 존재하나 플랫폼 수수료 별도 가능).
- **적용 방향**: 라이선스 확보 전까지 **NIV 비활성**. 확보 시 API.Bible 연동 + 본문 캐시(`verse` 캐시 정책은 API.Bible 약관 준수).
