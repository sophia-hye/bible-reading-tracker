# Bible Reading Tracker 기획 문서

> **요약**: 아이폰(iOS)과 갤럭시(Android)에서 모두 사용 가능한 성경 통독 진행률 추적 앱.
> 성경 1독 체크를 핵심으로, 진행률·통계·성취를 **시각적으로 "완성해 나가는 느낌"**으로 제공하고,
> 챌린지·친구 공유·교회 그룹 통독·리더보드 등 함께 읽는 동기부여 기능까지 포함한다.
> 여러 기기 간 로그인 동기화 지원.
>
> **Project**: Bible Reading Tracker

## 1. 개요 (Overview)

### 1.1 목적 (Purpose)

성경 통독을 꾸준히 이어가기 어렵다는 문제를 해결한다. 사용자가 매일 읽은 분량을 간편하게 체크하고,
진행률·연속 기록(streak)·통계·성취 배지를 **시각적으로 채워지는 형태**로 확인하여 통독 습관을 지속하게 돕는다.
나아가 혼자가 아니라 **친구·교회 그룹과 함께** 읽으며 서로 격려하는 환경을 만들어 완독률을 높인다.
여러 기기(아이폰/갤럭시/웹)에서 동일한 진행 상황을 동기화한다.

### 1.2 핵심 가치 제안 (Value Proposition)

| 키워드 | 설명 |
|--------|------|
| **완성감 (Sense of Completion)** | 1,189장이 하나씩 채워지는 시각화로 "내가 끝까지 가고 있다"는 성취감 제공 |
| **간편함 (Effortless)** | 탭 한 번으로 읽음 체크, 자동 진행률 계산, 오늘 읽을 분량 추천 |
| **동행 (Together)** | 친구·교회 그룹과 함께 읽고 격려, 리더보드로 선한 동기부여 |
| **꾸준함 (Consistency)** | streak, 알림, 챌린지, 배지로 매일 돌아오게 만드는 루프 |

### 1.3 배경 (Background)

- 기존 성경 앱은 읽기(본문) 기능 중심이고 "통독 진행 관리"는 부가 기능에 그치는 경우가 많음
- 종이 통독표는 기기 분실/중복 체크 어려움, 통계·동기부여 부재
- 통독은 혼자 하면 중도 포기율이 높음 → 교회/소그룹의 **함께 읽기**가 완독률을 크게 높임
- 가족/소그룹이 여러 기기를 쓰는 환경에서 진행률 공유 필요 → 로그인 기반 클라우드 동기화 요구

### 1.4 페르소나 (Personas)

| 페르소나 | 상황 | 핵심 니즈 |
|----------|------|----------|
| 개인 통독러 (P1) | 올해 처음 1독에 도전 | 매일 분량 추천 + 진행률 시각화 + 알림 |
| 교회 소그룹 리더 (P2) | 청년부 20명과 90일 통독 진행 | 그룹 생성, 진행 현황 한눈에, 독려 |
| 함께읽는 친구 (P3) | 친구와 서로 자극받으며 읽음 | 친구 추가, 리더보드, QT 인증 공유 |
| 복귀 독자 (P4) | 중간에 멈췄다가 다시 시작 | 어디까지 읽었는지 즉시 파악, 재시작 추천 |

## 2. 범위 (Scope)

> 기능이 커진 만큼 **단계별(MVP → Phase 2 → Phase 3)** 출시 전략을 적용한다.

### 2.1 MVP (1차 출시 — 개인 통독 핵심)

- [ ] 성경 66권 / 1,189장 기준 읽기 진행률 추적 (장별 체크/해제)
- [ ] **앱 내 성경 본문 읽기** — 개역한글(KRV) 우선 제공, NIV는 라이선스 확보 후 추가 (번역본 전환)
- [ ] **시각적 진행 트래킹**: 책별 진행 바, 전체 그리드(1,189칸) 히트맵, 원형 진행 게이지
- [ ] 자동 진행률 계산 (전체 %, 책별 %, 신약/구약 구분)
- [ ] 성경 권별 통계 (완독 책 수, 책별 읽은 장/남은 장)
- [ ] 통독 계획 템플릿 선택 (1년 통독 / 90일 통독 / 신약 90일 등) 및 일자별 분량 자동 배정
- [ ] **통독 순서 추천** — 카테고리(성경 순서 / 주제별 / 신약 먼저 등) 선택 후 사용자 지정
- [ ] **오늘 읽을 분량 추천** (선택한 계획 + 현재 진행 기반)
- [ ] 일일 읽기 체크 + 연속 읽기 기록(streak) + 전체 진행률(%)
- [ ] 성취 데이터 / 배지 (첫 1장, 7일 연속, 첫 책 완독, 신약 완독, 1독 완료 등)
- [ ] 매일 읽기 리마인더 푸시 알림 (시간 설정 가능)
- [ ] 회원가입 / 로그인 (이메일, Apple, Google)
- [ ] 여러 기기 간 자동 동기화 (오프라인 우선 + 온라인 동기화)
- [ ] 오프라인 사용 (네트워크 없이 체크 → 재연결 시 동기화)
- [ ] 다크 모드 / 한국어·영어 UI

### 2.2 Phase 2 (함께 읽기 — 소셜)

- [ ] 친구 추가 및 진행 기록 공유 (진행률/streak 카드 공유)
- [ ] 교회/소그룹 통독 그룹 생성·참여 (초대 코드/링크)
- [ ] 그룹 진행 현황 대시보드 (멤버별 진행률, 그룹 누적 달성률)
- [ ] 90일 통독 챌린지 / 365일 성경 읽기 챌린지 (기간형 챌린지)
- [ ] 리더보드 (공개 범위 선택: 개인용 / 그룹 공개 / 전체 공개(public))
- [ ] QT(묵상) 메뉴 — **진입점·플레이스홀더만 배치, 상세 기획은 추후 보강**

### 2.3 Phase 3 (확장 — 향후 검토)

- [ ] 그룹 채팅/응원 댓글, 격려 이모지(리액션)
- [ ] 맞춤형 통독 계획 직접 생성 (시작일/기간/순서 커스텀)
- [ ] 통계 심화 (시간대별 읽기 습관, 연도별 회고 리포트)
- [ ] 위젯 / 잠금화면 진행률, Apple Watch·Wear OS 연동
- [ ] 결제 / 프리미엄 (광고 제거, 고급 통계 등)

### 2.4 Out of Scope (제외)

- NIV 본문 제공 — **라이선스(API.Bible/Biblica) 확보 전까지 제외**, 확보 후 추가 (개역한글은 MVP 포함)
- 오디오 성경 재생
- 공개 SNS 피드(전체 공개 타임라인) — 단, 리더보드의 public 옵션은 예외(2.2 참조)
- 결제 / 유료 구독 (Phase 3 이후 검토)

## 4. 핵심 UX — 시각적 완성감 설계 (Visual Completion UX)

> 사용자가 "끝까지 채워가는 느낌"을 받도록 진행을 **다층적으로 시각화**한다. 이것이 본 앱의 차별점.

### 4.1 진행 시각화 컴포넌트

| 컴포넌트 | 설명 | 위치 |
|----------|------|------|
| **전체 원형 게이지** | 1,189장 대비 읽은 장 비율을 큰 원형 링으로 표시 (% + 남은 장) | 홈 상단 |
| **66권 그리드 / 책 타일** | 각 책을 타일로, 완독 시 채색·체크 표시 (구약 39 / 신약 27 구분) | 진행 탭 |
| **1,189칸 히트맵** | 장 단위 미니 셀 그리드, 읽을수록 채워짐 (GitHub 잔디 느낌) | 진행 탭 상세 |
| **책별 진행 바** | 책 안에서 읽은 장 / 전체 장 막대 | 책 상세 |
| **streak 불꽃/캘린더** | 연속 읽은 날 캘린더 히트맵 + 연속일 카운터 | 홈 |
| **배지 선반** | 획득한 성취 배지 진열, 미획득은 실루엣 | 성취 탭 |
| **마일스톤 진행 타임라인** | 1독 여정의 큰 이정표(오경 완독→신약 시작→완독) | 성취 탭 |
| **비례 점 매트릭스 통독표** | 행=책·열=장, 점 크기=장 길이, 읽은 장은 잉크 검정 (스위스 스타일, 4.3) | 진행 탭(장르별) |

### 4.2 피드백·연출

- 장 체크 시 마이크로 인터랙션(채워짐 애니메이션 + 햅틱)
- 책 완독/배지 획득 시 축하 모달(컨페티, 공유 버튼)
- 진행 농담(명도)은 누적에 따라 단계적 변화로 "성장" 체감 (무채색 톤 유지, 4.3 참조)

### 4.3 비례 점 매트릭스 통독표 (Proportional Dot Matrix) — 핵심 아트 컨셉

> 스위스 스타일 **비례 점 매트릭스(proportional dot matrix)** 인포그래픽을 통독표에 적용.
> **행 = 성경 책, 열 = 장 번호**, **점 크기 = 장 길이(절 수)**, 읽은 장은 잉크 검정으로 채운다.
> 무채색·여백 중심의 미니멀 에디토리얼 미학으로 브랜드(무채색·미니멀)와 일치.
>
> **레퍼런스 미학**: ① 행=카테고리 / 열=인덱스 매트릭스, ② 점 크기로 양적 정보 인코딩,
> ③ 극도의 흑백 미니멀 + 충분한 여백, ④ 작은 산세리프 라벨(좌측 행명·하단 열번호).

**적용 매핑**

| 레퍼런스 | 통독표 적용 |
|----------|------------|
| 행(카테고리) | 성경 책 (책마다 장 수 달라 행 길이가 들쭉날쭉 → 책 길이가 한눈에) |
| 열(인덱스) | 장 번호 (하단 5·10단위 라벨) |
| 점 크기 = 데이터값 | **점 크기 = 장 길이(절 수)** — 큰 점=긴 장, 오해가 아니라 정보 |
| 점 채움 | 읽음 = 잉크 검정 / 안 읽음 = 연회색 |

**색/톤 전략 (확정): 무채색 모노톤 잉크**

- 통독표 배경은 **종이색(off-white)** 고정 → 잉크 점 대비 확보(라이트·다크 모드 무관)
- 읽은 장 = **잉크 검정**, 안 읽은 장 = **연회색** (이진). 다색은 쓰지 않음
- 진행은 **검정 채움이 행 단위로 번지며** 표현 → 한 책을 다 읽으면 그 행이 검게 완성
- (선택) 완성 임박 시 농담에 방향성을 약하게 더해 "그라데이션 흐름"을 줄 수 있음 — 구현 단계 미세조정

**화면 단위: 장르별**

오경/역사서/시가서/대선지서/소선지서(구약) + 복음서/역사서(행)/바울서신/일반서신/예언서(신약) 단위로 한 매트릭스.
(현 Figma 'Tracker' 디자인의 장르별 분할과 동일 — 시편 150장처럼 긴 책도 폰 화면에 수용)

**설계 원칙**

| # | 원칙 | 이유 |
|---|------|------|
| 1 | **행=책 / 열=장 매트릭스** (장르별 화면) | 성경 구조 그대로 매핑, 진행·위치 파악 즉시 |
| 1b | **모바일은 책별 장을 N열(기본 10)로 줄바꿈(wrap)** | 시편(150장) 등 긴 책의 **가로 스크롤 방지** → 세로 스크롤. 모바일 10열 / 태블릿·가로 15~20열 (반응형) |
| 2 | **점 크기 = 장 길이(절 수) 인코딩** | 레퍼런스 정신, 크기가 정보가 됨. 데이터 기반이라 항상 동일(결정론적) |
| 3 | **5·10장 단위 열 번호 라벨** (또는 wrap 줄 자체가 10장 눈금) | 장 번호를 빠르게 세고 위치 찾기 |
| 4 | **읽음 = 잉크 검정 / 안 읽음 = 연회색**, 종이색 배경 | 무채색 미니멀, 채움이 행 단위로 번짐 |
| 5 | **면적 착시 보완** | 점 크기는 장 길이일 뿐 진행량 아님 → 진행률 %·읽은 장 수를 숫자로 병기 |
| 6 | **터치는 시각과 분리 (히트박스 확대)** | 짧은 장(작은 점)도 투명 탭 영역을 크게 → 접근성·사용성 |
| 7 | (선택) **완독 포스터/공유 카드** | 다 채운 검은 매트릭스에 여백·타이포 얹어 소장·공유용 작품화 |

**모드 분리(선택지)**: 트래킹 가독성이 더 필요하면 "균일 그리드(1,189칸 히트맵)"와 "매트릭스 아트 모드"를 토글로 제공.

**절(verse) 단위 정밀 트래킹 (확정 방향)**: 장 단위가 거칠어 **절까지** 체크.
- **앱 = 드릴다운**: 장 보기 점 → 한 장 탭 시 그 장의 **절 그리드**(1 dot = 1 verse)로 진입해 절 단위 체크.
- **인쇄 = 절 점 + 장 구분**: 1 dot = 1 verse, 장별로 묶어 표시(무채색, 1도 인쇄).
- **전제 데이터**: 장별 절 수 데이터셋(성경 총 약 31,102절) 필요 → 현 시안은 근사값, 실제 적용 시 정확 데이터로 교체.
- **번역본별 절 수 차이 + 온보딩 선택**: 개역한글과 NIV는 절 구분이 달라 절 수가 다름(예: 개역한글 ~31,102 / NIV ~31,086). **첫 설정(온보딩 Step 1)에서 번역본을 선택**하고, 그 번역본의 절 수 데이터(`verse.versionId`별)로 절 동그라미 개수를 생성. 설정에서 변경 가능. (MVP=개역한글, NIV=라이선스 후 "곧 제공")
- (대안) 장 점을 절 비율로 부분 채움하는 방식(A안)도 가능 — 점 1,189개 유지하면서 절 정밀도.

> 디자인 참조: 사용자 제시 레퍼런스(스위스 비례 점 매트릭스 인포그래픽) + Figma 'Tracker' 페이지 — [[figma-tracker-design]] (장르별 장 그리드 + 절 단위 드릴다운/인쇄 시안, 흑백 미니멀).

## 5. 게이미피케이션 & 동기부여 (Gamification)

### 5.1 성취 / 배지 (Achievements) 예시

| 배지 | 조건 | 분류 |
|------|------|------|
| 첫 걸음 | 첫 1장 읽기 | 입문 |
| 한 주 완주 | 7일 연속 streak | 습관 |
| 꾸준함의 달인 | 30일 연속 streak | 습관 |
| 첫 책 완독 | 한 권 전체 완독 | 진행 |
| 오경 정복 | 창세기~신명기 완독 | 진행 |
| 복음서 통과 | 4복음서 완독 | 진행 |
| 신약 완독 | 신약 27권 완독 | 진행 |
| 통독 완성 | 성경 1독 완료 | 정상 |
| 90일 챌린저 | 90일 챌린지 완주 | 챌린지 |
| 함께의 힘 | 그룹 통독 완주 | 소셜 |

### 5.2 챌린지 (Challenges)

| 챌린지 | 기간 | 일 분량(개략) | 비고 |
|--------|------|--------------|------|
| 90일 통독 | 90일 | 약 13~14장/일 | 집중 통독 |
| 365일 성경 읽기 | 1년 | 약 3~4장/일 | 가장 보편적 |
| 신약 90일 | 90일 | 약 3장/일 | 입문자용 |
| 시편/잠언 한 달 | 30일 | 약 5장/일 | 단기 도전 |

- 챌린지는 개인 단독 또는 그룹 단위 참여 가능
- 진행 상황은 챌린지 전용 진행 바 + 리더보드로 표시

### 5.3 리더보드

- **공개 범위(사용자 선택)**:
  - **개인용(Private)** — 나만 보는 개인 기록/순위 (기본값)
  - **그룹 공개(Group)** — 내가 선택한 특정 그룹에만 공개
  - **전체 공개(Public)** — 앱 전체 공개 리더보드에 노출
- 기준(선택): 누적 읽은 장, 현재 streak, 챌린지 진행률
- 정책: **경쟁보다 격려** — 기본값은 비공개(Private), 공개는 명시적 옵트인, 하위권 압박 최소화(예: 본인 ±N명만 표시 옵션)

### 5.4 일일 복귀 루프 (Retention Loop)

알림 → 오늘 분량 추천 → 탭 체크(완성감 연출) → streak·배지 업데이트 → 친구/그룹 활동 확인

## 6. 추천 로직 — 통독 순서 & 오늘 읽을 분량 (Recommendation)

### 6.0 통독 순서 카테고리 (Reading Order)

사용자가 **카테고리를 선택**하면 그에 맞는 읽기 순서/계획을 추천하고, 이후 **사용자가 직접 조정(사용자 지정)** 가능.

| 카테고리 | 설명 | 예시 순서 |
|----------|------|----------|
| 성경 순서 (정경순) | 창세기→요한계시록 그대로 | 전통적 통독 |
| 신약 먼저 | 신약 27권 후 구약 | 입문자/복귀자 추천 |
| 주제별 | 주제(은혜/지혜/예언/복음 등) 묶음으로 | 시편·잠언, 4복음서 등 테마 |
| 역사 흐름순 | 사건 연대기 순서 | 역사 이해 중심 |
| 장르별 | 율법/역사서/시가서/예언서/복음서/서신서 | 문학 유형별 |

- 카테고리 선택 → 추천 계획 미리보기 → 시작일·기간 조정 → 확정
- 확정된 순서는 `user_plan.schedule`로 저장, 오늘 분량 추천(6.1~6.3)의 입력이 됨

### 6.1 입력

- 선택한 통독 계획(총 일수, 일자별 분량표) 또는 챌린지
- 현재까지 읽은 장 / 마지막 읽은 위치
- 계획 시작일·오늘 날짜 (밀린 분량 계산)

### 6.2 규칙(초안)

1. **계획 기반**: 계획표의 "오늘 해당 일자" 분량을 기본 추천
2. **따라잡기(catch-up)**: 밀렸다면 남은 일수에 남은 분량을 재분배해 "오늘 권장량" 산출(과도하면 상한 적용)
3. **앞서감(ahead)**: 앞서 있으면 "오늘은 가볍게 / 다음 분량 미리읽기" 옵션 제시
4. **계획 미선택 시**: 균등 분배(예: 1년=하루 약 3.3장) 기반 다음 미읽은 구간 추천

### 6.3 출력

- "오늘 읽을 분량" 카드 (예: 마태복음 5~7장) + 예상 소요 시간 + 바로 체크 버튼

## 7. 데이터 모델 개요 (Data Model — 초안)

> 상세 스키마는 설계 단계에서 확정. 오프라인(SQLite) ↔ 서버(BaaS) 동기화 대상.

| 엔티티 | 핵심 필드 | 설명 |
|--------|----------|------|
| `user` | id, email, displayName, photo, locale, preferredVersion, leaderboardVisibility(private/group/public), createdAt | 계정 + 기본 번역본·리더보드 공개 설정 |
| `book` (정적) | id, name, testament, order, chapterCount, genre | 성경 66권 메타 (장르=율법/역사/시가/예언/복음/서신) |
| `bible_version` (정적) | id, code(KRV/NIV), name, source(local/api), licenseNote | 번역본 메타 (개역한글=로컬 번들, NIV=API.Bible) |
| `verse` (정적/캐시) | versionId, bookId, chapter, verse, text | 본문 (개역한글 번들; NIV는 라이선스 후 API 캐시) |
| `reading_log` | id, userId, bookId, chapter, readAt, source, updatedAt | 장별 읽음 기록(체크) |
| `reading_plan` (정적/템플릿) | id, name, totalDays, schedule(JSON) | 통독 계획 템플릿 |
| `user_plan` | id, userId, planId, startDate, status | 사용자가 선택한 계획 |
| `streak` | userId, current, longest, lastReadDate | 연속 기록 |
| `achievement` | id, userId, badgeKey, achievedAt | 획득 배지 |
| `challenge` | id, type, title, startDate, endDate, rule | 챌린지 정의 |
| `challenge_member` | challengeId, userId, progress, joinedAt | 챌린지 참여 |
| `group` | id, name, ownerId, inviteCode, planId | 교회/소그룹 |
| `group_member` | groupId, userId, role, joinedAt | 그룹 멤버 |
| `friendship` | userId, friendId, status | 친구 관계 |
| `qt_post` | id, userId, scope, text, imageUrl, bookRef, createdAt | QT/묵상 인증 |
| `sync_meta` | entity, lastSyncedAt, deviceId | 동기화 메타 |

> 동기화 정책: 서버 타임스탬프 기준 last-write-wins, `reading_log`는 멱등(idempotent) 업서트(같은 user+book+chapter 1건).

## 10. 아키텍처 고려사항 (Architecture Considerations)

### 10.1 프로젝트 레벨 선택

| Level | 특징 | 적합 대상 | 선택 |
|-------|------|----------|:----:|
| **Starter** | 단순 구조, 백엔드 없음 | 정적 사이트, 포트폴리오 | ☐ |
| **Dynamic** | 기능 모듈 구조 + BaaS 통합 | 백엔드 있는 앱, SaaS MVP | ☑ |
| **Enterprise** | 엄격한 레이어 분리, 마이크로서비스 | 대규모 트래픽 | ☐ |

> **선택: Dynamic** — 로그인 + 멀티 기기 동기화 + 소셜(그룹/리더보드)이 필요하므로 BaaS 기반 풀스택 구조가 적합.

### 10.2 핵심 아키텍처 결정

| 결정 항목 | 옵션 | 선택 | 근거 |
|----------|------|------|------|
| 앱 프레임워크 | React Native(Expo) / Flutter / PWA | **React Native + Expo** | 코드 하나로 iOS/Android, 푸시·오프라인 강점, TS 생태계 |
| 언어 | TypeScript / JavaScript | **TypeScript** | 타입 안정성 |
| 상태관리 | Context / Zustand / Redux | **Zustand** | 경량, 보일러플레이트 적음 |

| 백엔드(BaaS) | Supabase / Firebase / bkend.ai | **확정: Supabase** | Postgres + Auth + 실시간(그룹/리더보드) + RLS, 무료 티어 |
| 인증 | BaaS Auth (이메일+소셜) | **BaaS 내장 Auth** | 구현 비용 절감 |

> 본 조사는 공개 자료 기반 1차 검토이며, **실제 적용 전 각 기관에 서면 확인 필요**.

### A.1 개역한글 (KRV) — 대한성서공회

- **저작권료**: 무료. 저작재산권 보호기간(50년) 경과로 저작권료 지급 없이 사용 가능.
- **준수 조건 (인격저작권)**:
  - **동일성유지권**: 본문 내용·형식·제호 무단 변경/삭제/개변 불가 → 앱에서 본문을 **수정 없이 원문 그대로** 제공해야 함.
  - **성명표시권**: 저작자/출처 표기 준수.
- **앱/디지털 사용**: 공개 FAQ에 디지털·앱 별도 허가 여부 명시 없음 → **대한성서공회(02-2103-8700)에 앱 탑재 허가·표기 방식 사전 확인 권장**.
- **적용 방향**: MVP에 로컬 번들로 탑재(데이터셋 확보 필요), 출처 표기 + 본문 불변 보장.

### A.2 NIV — Biblica / API.Bible

- **저작권**: 보호 대상. 무단 사용 불가.
- **라이선스 경로**: **API.Bible**(scripture.api.bible)를 통해 번역본 단위 라이선스.
  - 비상업 Starter: 무료(월 5,000 콜, 일부 번역본).
  - 상업적 사용: 번역본당 **월 $10~** + Pro 플랜(월 150,000 콜 수준). "상업적 사용"에는 광고·인앱·구독·유료 접근 등 수익화 일체 포함.
  - 비인쇄/디지털 상업 사용은 **Biblica 별도 허가 신청** 필요(일부 royalty-free 라이선스 존재하나 플랫폼 수수료 별도 가능).
- **적용 방향**: 라이선스 확보 전까지 **NIV 비활성**. 확보 시 API.Bible 연동 + 본문 캐시(`verse` 캐시 정책은 API.Bible 약관 준수).
